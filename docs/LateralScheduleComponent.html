Name Type Description
tarjetaPaciente string Retorna los datos del paciente y la fecha y hora de inicio y final de la consulta (solo en consulta editada)

# static watch

Variables a observar por el componente
Properties:
Name Type Description
startTime string Al actualiar la hora de inicio se actualiza la fecha y hora de inicio de la cita
finishTime string Al actualiar la hora de finalización se actualiza la fecha y hora de finalización de la cita
patientID number Al actualizar el ID del paciente logueado actualemente, se asigna a la variable scheduleSelectedCopy.patient_id el nuevo ID del paciente
dateSelected string Al actualiar la fecha de la consulta se actualiza la fecha y hora de la cita
schedule Schedule Al actualizar la cita seleccionada, se obtiene el doctor seleccionado, se asigna un copia de la consulta a la variable scheduleSelectedCopy, se asigna a la variable dateSelected la fecha de la consulta. Si el usuario logueado actualmente es un paciente se asigna a la variable scheduleSelectedCopy.patient_id el ID del paciente. Si la lista de doctores (doctorsList) tiene datos, se asigna una copia a la variable doctorListCopy, en caso contrario se obtiene la lista de doctores. Finalmente selecciona y muestra en el input del doctor, el doctor seleccionado
doctorListCopy Array.<Select> Si se actuliza la copia de los doctores de la sucursal seleccionada, se muestra en el input del doctor, el doctor seleccionado
scheduleSelectedCopy.branch_id number Si se actualiza el ID de la sucursal de la cita seleccionada, verifica si la lista de los doctores (doctorsList) es igual a su copia (doctorsList), además verifica si el ID de la sucursal de la consulta (schedule.branch_id) es igual que la copia de la consulta (scheduleSelectedCopy.branch_id): en caso de se diferente alguno de las dos verificaciones obtiene la lista de doctores
scheduleSelectedCopy.doctor_id number Si se actualiza el ID del doctor, verifica el tipo de doctor seleccionado, en caso de ser laboratorio o imagenologia, se obtiene la lista de estudios médicos, además que se asigna a la variable scheduleSelectedCopy.medicalconsultcategory_id el tipo de consulta médica a la que pertenece la consulta
doctorSelect number Al actualizar el doctor seleccionado, se asigna a la copia de la consulta el ID del doctor (scheduleSelectedCopy.doctor_id) y la especialidad médica a la que pertenece (scheduleSelectedCopy.medicalspecialty_id)

# static getDoctorPosition(doctorsList, doctorID, specialtyID)

Obtiene la posición del doctor dentro de la variable doctorsList de acuerdo a la especialidad médica a la que pertenece.
